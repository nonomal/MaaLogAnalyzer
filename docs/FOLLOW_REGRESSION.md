# Follow 回归测试（基于内置教程样例）

本文用于快速回归日志分析页的 `Follow` 行为。

## 使用样例

项目内置样例：`src/assets/tutorial-sample.log`。

无需手动找文件，直接通过 UI 加载：

1. 打开关于页。
2. 点击 `开始新手教程`。
3. 样例会自动加载到分析页。
4. 如果不需要继续教程，可点 `跳过`（样例数据会保留）。

## 回归步骤

1. 进入 `日志分析` 视图（桌面端）。
2. 确认标题右侧同一行显示：`跟随中/跟随最新/未实时` + `重新加载`。
3. 确认正文顶部不再有单独的“操作按钮”空白区。
4. 点击任务列表任意旧任务，确认跟随状态从 `跟随中` 变为 `跟随最新`。
5. 再点击 `跟随最新`，确认回到 `跟随中` 并自动定位到最新任务/最新节点。

## 实时压测（可选）

在控制台执行已有 mock 脚本（`__maaRtMock.start(80)`），观察：

1. 页面持续跟随最新任务与节点。
2. 不出现 `Cannot read properties of undefined (reading 'accumulator')`。
3. 不出现明显闪回、跳到错误任务或卡住不动。

## 通过标准

1. Follow 可在实时流中稳定工作。
2. 手动切换历史任务后不会被立即强制抢回，除非再次开启跟随。
3. 头部操作区布局稳定，无重复按钮。
